# ATTENTION #
For this development boilerplate setup to work properly after first downloading it, the following files must be created locally:
- `.env`
  
The file must contain the following environment variable definition:
`WEBPACK_ENV=production`

(`production`is just a placeholder. The variable will be set appropiately once the app is run by means of the corresponding scripts present in the `package.json` file - `build` for production purposes; `dev` for development)
